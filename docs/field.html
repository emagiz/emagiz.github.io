id: title: FieldField

XML tag

Tag name for this field's XML element.

Required

Documentation

Documentation describing this field.

Optional

Has components

Whether this field consists of (multiple) components, or contains just a single value.

Required

Whether the value for this field is required or not.

Default is false.

Data type

The XML Schema data type of this field.

This data type is only used for data validation, not for data conversion: if the EDI data doesn't exactly correspond to any of the XML Schema formats, use xs:string instead (or just don't specify any type at all).

Optional

Min length

The minimum number of characters allowed in the value.

This data type is only used for data validation, not for data conversion (i.e. data won't be padded to fit).

Optional

Max length

The maximum number of characters allowed in the value.

This data type is only used for data validation, not for data conversion (i.e. data won't be truncated to fit).

Optional

Truncatable

Whether it's allowed to omit trailing empty components (that are not required) for this field.

Note that components that are required can never be omitted.

Default is false.

The different components that can appear in this field, in the order specified.

In the following example, the ADDRESS segment consists of the fields street, town and postcode (field separator is a semicolon), where the street field consists of the components number and street name (component separator is a colon):

ADDRESS;3:High Street;SOUTHAMPTON;SO31 4NG
