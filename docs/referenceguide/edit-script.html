SQL Script

SQL script to execute to populate, initialize, or clean up the database.

Location

The resource location of an SQL script to execute. Can be a single script location or a pattern (e.g. classpath:/com/foo/sql/*-data.sql

When using a H2 database for a Spring Integration (channel) message store use this location: classpath:com/emagiz/components/jdbc/int-schema-h2.sql

When using a H2 database for Spring Batch (ETL) use this location: classpath:com/emagiz/components/jdbc/batch-schema-h2.sql

To use custom SQL scripts, add them to the flow resources and then use the prefix resource/ in front of the filename. Uploading the script will change the filename. To find the new filename download the file after uploading it.

Encoding

The encoding for SQL scripts, if different from the platform encoding.

Separator

The default statement separator to use. The default is to use ; if it is present in the script, or \n (new line) otherwise.

Execution

Indicate the execution phase of this script. Use Init to execute on startup (as a bean initialization) or Destroy to execute on shutdown (as a bean destruction callback).

Default is Init.
