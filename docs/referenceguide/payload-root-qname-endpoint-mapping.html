Payload root QName endpoint mapping

Maps request messages to endpoints, based on the name of the root XML element.

Uses the qualified name of the root element of the webservice request payload to determine the endpoint that handles it.

Default endpoint

The default endpoint to use, when this endpoint mapping does not result in a matching endpoint.

The mappings between qualified root names and endpoints

Interceptors that will be applied in the specified order to the request and response message.

Available types:

Payload validating interceptor Interceptor that validates the contents of web service messages using a schema. If not valid, the processing of the message will be stopped.

HTTP request address extracting interceptor Interceptor that extracts the remote and local IP address and port from the incoming HTTP request, so these values can be used later in the message flow.

WSS4J security interceptor Interceptor that can validate WS-Security information in requests and can secure the responses.
