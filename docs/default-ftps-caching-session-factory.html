default-ftps-caching-session-factory

Default FTPS caching session factory

Session factory for creating connections with an FTPS server that uses session caching.

Documentation

FTPS session factory that provides session caching/pooling, re-using existing sessions for multiple request.

When used by a FTP inbound channel adapter this only creates a new session + login attempt each time the adapter synchronizes the remote FTPS directory and no idle cached session is available. When used by a FTP outbound channel adapter this only creates a new session + login attempt each time the adapter receives a message and no idle cached session is available.

Id

Name that uniquely identifies this flow component.

Required

Session cache size

Controls how many active sessions this adapter will maintain in its cache. If the session cache size threshold has been reached, any attempt to acquire another session will block until either one of the cached sessions becomes available or until the wait time for a session expires.

Default is unbounded (i.e. an unlimited amount of active sessions is allowed).

Session wait timeout

The limit of how long to wait (in milliseconds) for a session to become available.

Default is Integer.MAX_VALUE (i.e. wait indefinitely for a session to become available).

Protocol

The secure socket protocol to be used, e.g. SSL/TLS.

Default is TLS.

Implicit

The security mode: true: implicit mode false: explicit mode

Default is false (explicit mode).

Session creation

Controls whether a new SSL session may be established by this socket.

Default is true.

Use client mode

Configures the socket to use client (or server) mode in its first handshake.

Default is true (client mode).

Need client auth

If true, configures the socket to require client authentication.

Default is false (client authentication not required).

Wants client auth

Configures the socket to request client authentication, but only if such a request is appropriate to the cipher suite negotiated.

Default is false (don't request client authentication).

AUTH value

Set AUTH command use value. This processing is done before connected processing.

Default is TLS.

Cipher suites

Comma-separated list of cipher suites. Controls which particular cipher suites are enabled for use on this connection. Called before server negotiation.

Default is null (no particular cipher suites enabled).

PROT

Data channel protection level (PROT command): C - Clear S - Safe (SSL protocol only) E - Confidential (SSL protocol only) P - Private

Default is P (private).

Protocols

Comma-separated list of protocol versions. Controls which particular protocol versions are enabled for use on this connection.

Default is null (no particular protocols enabled).

File type

Indicates how the file(s) being transfered should be treated: 0 - ASCII file type 1 - EBCDIC file type 2 - Binary file type 3 - Local file type

Default is 2 (binary file type, i.e. no translations should be performed).

Buffer size

Default is '2048'.

see https://issues.apache.org/jira/browse/NET-207

Client mode

Indicates how data transfers are expected to occur between the client (local) and server (remote). Choices are:

0 - Active local data connection mode The server should connect to the client's data port to initiate a data transfer. Note that these types of connections are problematic when the server is not located in the same network, because firewalls on the client side usually block incoming connections.

2 - Passive local data connection mode The server is in passive mode, requiring the client to connect to the server's data port to initiate a transfer.

Default is 0 (active mode).

Control encoding

Sets the character encoding used by the FTP control connection. Some FTP servers require that commands be issued in a non-ASCII encoding like UTF-8 so that filenames with multi-byte character representations (e.g, Big 8) can be specified.

Default is ISO-8859-1.

Connect timeout

The socket timeout (in milliseconds) when connecting to the server.

A timeout of zero is interpreted as an infinite timeout: the connection will block until established or an error occurs.

Default is 0 (infinite timeout).

Default timeout

The socket timeout (in milliseconds) when reading data from an established connection.

A timeout of zero is interpreted as an infinite timeout: the connection will block until the data has been read or an error occurs.

Default is 0 (infinite timeout).

Data timeout

The socket timeout (in milliseconds) when reading from the data connection.

A timeout of zero is interpreted as an infinite timeout: the data connection will block until the data has been read or an error occurs.

Note that this timeout will also be applied when accepting incoming connections whilst establishing an active local data connection.

Default is 0 (infinite timeout).
