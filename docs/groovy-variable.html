groovy-variable

Groovy variable

Allows you to define custom script variable bindings. This attribute isn't mutually exclusive with variables setting (see the basic tab) and all variables will be merged to one map.

The variables payload and headers are always automatically available, even if you do not manually specify any variables. Since these two variables already give you access to all information in the message from your Groovy script, the main use-case for custom variables is probably to pass property values to the script.

_id

Value for this variable as a reference to any bean in this flow. This can be useful to pass a support object to the Groovy script, for example passing a composite file list filter if the Groovy script wants to re-use the file filtering functionality.

Either value or ref must have a value (but not both).
