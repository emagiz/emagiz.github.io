transform-from-string

id: title: Transform from stringTransform from string

Starting position for the substring. The first character has position 1, the second position 2, etc.

Example: with start position 2, the string eMagiz will be transformed to Magiz.

Where to start counting the characters to find the start position. When selecting from end the last character has position 1, the second to last character has position 2, etc.

Example: with start position 2 from end, the string eMagiz will be transformed to iz.

Length of the substring to return. If empty, this returns the substring from the start position to the end of the string.

Example: with start position 2 and length 4, the string eMagiz will be transformed to Magi.

All occurrences of characters in the input string that match this regular expression will be replaced.

Example: with regex [aeiou] and an empty replacement value, the string eMagiz will be transformed to Mgz.

See also: W3C regular expression syntax

String to replace all characters that matched the regular expression with. Leaving this empty will replace them with an empty string.

Can also contain backreferences to capturing groups in the regular expression by using the $n notation, where 'n' is the group number.

Example: with regex (.) - (.) and replacement $2 - $1, the string eMagiz - Platypus will be transformed to Platypus - eMagiz.

See also: W3C XPath functions reference

Transformer with multiple options for converting string values.

Transformer with multiple options for converting string values.

Option substring allows you to extract part of a value, for example select only the first 10 characters of a longer string value.

With regex replace you can perform complex find-and-replace operations on string values.

With convert datatype you can convert a string input to another data type. Currently it is only possible to convert to date/time data types.

Selecting custom XPath allows you to fully control the transformation of any value by entering your own custom XPath expression to handle the conversion. Technically, the XPath you enter will be used for the <xsl:value-of> statement in the generated stylesheet. Use the view XSLT button to see a preview how your XPath will be embedded in the resulting XSLT.

Time zone format

The time zone to use when converting the input string. If the input string has time zone information, you can preserved it by leaving this field empty, otherwise it will overwrite the zone information with the selected value.

Pattern to use for converting the string value into a date/time data type. The pattern is created based on your date, time, timezone, and delimiters selection. You can create your own custom format by using the generated pattern as a starting point.

More information about the pattern format syntax can be found in the Documentation
